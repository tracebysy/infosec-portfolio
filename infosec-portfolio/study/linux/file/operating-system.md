# 개요
하드웨어와 스프트웨어를 사용자가 편하게 사용할 수 있는 환경을 제공하는 프로그램으로 컴퓨터 시스템의 자원을 효율적으로 할당, 관리, 보고 한다.
사용자의 입력을 받아 컴퓨터가 이해하는 언어나 방식으로 해석하여 컴퓨터를 동작시키고, 결과를 받아 사용자가 이해할 수 있는 방식으로 도출한다.

- GUI
: 모든 입출력, 처리 -> 색, 모양, 형태 등의 그래픽으로 보여주거나 처리
👍 장점
(1) 편의성이 좋다.
(2) 직관적이다.
(3) 사용자 친화적이다.
ex) 마우스
👎 단점
(1) 그래픽을 처리하는데 자원을 많이 소모 한다.
(2) 속도가 상대적으로 느리다
ex) 주로 PC에서 사용

- CLI
: 명령어를 기반으로 입출력을 처리
👍 장점
(1) 자원의 소모가 적다.
(2) 컴퓨터 부담이 적다.
(3) 속도가 빠르다
👎 단점
(1) 편의성이 낮다.
(2) 명령어를 모르면 컴퓨터 사용이 불가능하다. -> 비사용자 친화적
ex) 주로 SERVER에서 사용

---

## 운영체제 구성 요소
1. user interface
- 사용자에게 명령 입력을 받는 프로그램으로
데이터 처리 결과를 사용자가 이해하기 쉽도록 출력하는 프로그램이다.
2. shell
- 명령어 해석기
- 사용자의 명령을 컴퓨터가 이해할 수 있는 언어나 방식으로 해석해주는 프로그램이다.
3. kernel
- 운영체제에서 핵심이 되는 코드나 제어 프로그램으로 하드웨어, 프로세스 등을 제어하는 자원 관리자이다.

## linux 파일의 종류
1. 일반 파일
- 기본적인 대부분의 파일 -> 음악, 실행, 영상, 문서 파일 등
2. 디렉터리 파일
- 일반 파일의 목록을 저장하고 있는 특구한 파일
3. 장치 파일
- 내/외부 장치를 하나의 파일로 인식하여 관리
4. 링크 파일
- 원본 파일에 대한 경로 정보나 내용을 저장하고 있는 파일 -> windows의 바로가기 파일과 비슷

## 디렉터리 구조
=> 역 트리 구조 ex) /root/home/gymin
<경로 : 정보 시스템의 자원이나 파일, 데이터를 찾아가거나 지정하기 위해 사용하는 주소 형태의 데이터로 절대 경로와 상대 경로가 있다.>
1. 절대 경로
- 최상위 디렉터리를 기준으로 경로 정보를 표현하여 사용자의 현재 위치와는 관계 없이 표현 할 수 있다.
2. 상대 경로
- 현재 위치를 기준으로 경로를 표현하기 때문에 현재 자신의 위치에서 원하는 파일의 위치까지 도달 할 수 있는 경로를 정확하게 알 고 있어야 한다.
