# 개요
리눅스에서 파일과 디렉토리를 생성, 조회, 복사, 삭제하기 위한 기본 명령어들이다.

---

## 주요 명령어

1. man + <명령어>  
: 명령어의 매뉴얼 확인  
-> 해당 명령어가 무슨 명령어 인지, 어떤 식으로 사용해야 하는지, 옵션은 뭐가 있는지 알 수 있음  
: 그만 보고 싶을 때는 ‘q’
2. date   
: 현재 시스템의 시간을 출력
3. hwclock  
: 현재 하드웨어의 시간을 출력
4. uname  
: 시스템의 정보를 확인  
: 시스템의 이름이나 버전 정보 등을 출력  
- uname -a → 주로 많이 사용
5. which + <명령어>  
: 해당 명령어가 저장 된 경로를 출력
6. clear  
: 프롬포트를 위로 올려 화면을 깨끗하게 함  
-> 지우는 건 아님  
: ctrl + l → 위로 올라간 명령어들을 다시 볼 수 있음
7. echo  
: echo 다음 입력한 글자를 표준 출력  
-> 리다이렉션 기호와 같이 사용  
‘<,>’리다이렉션 기호 → 표준 출력 데이터의 방향을 바꿈, 덮어쓰기
‘>>’ → 덮어쓰기가 아닌 이어쓰기
8. history  
: 현재 로그인한 목록, 입력했던 명령어 목록 출력  
: history + 숫자 → 최근 사용한 명령어를 숫자 만큼 출력
9. alias  
: 명령어 별칭을 만드는 명령어  
: 길고 까다로운 형태의 명령어를 간편하게 사용 → 주기적으로 반복해야 하는 업무 등에 사용
10. unalias  
: 별칭 명령어 해제
11. init 0  
: 리눅스 시스템 종료
12. shutdown + 옵션 + 시간 +(메시지)  
: 사용하는 옵션에 따라 시스템을 종료하거나 재부팅 하는 명령어  
<옵션>  
-h : 시스템 종료  
-r : 시스템 재부팅    
<시간>  
now : 지금 바로 종료나 재부팅  
(상대 시간 지정)숫자 - 몇 분 뒤에 종료나 재부팅  
(절대 시간 지정)00:00 - 지정한 시간에 종료 및 재부팅  
<메시지>  
현재 로그인한 사용자에게 공지로 전달  
13. power off  
: 시스템 강제 종료 명령어
14. halt  
: 시스템 종료 명령어 → 일부 동기화 과정이 있음
15. reboot  
: 시스템 재시작 명령어
16. init  
: 리눅스의 run level을 선택하는 명령어  
+ run level : 리눅스가 동작하는 모드  
>> run level
0 : 종료모드  
1 : 싱글모드(단일 사용자 모드) → 시스템 복구  
2 : CLI모드(다중 사용자 모드) → 네트워크 기능이나 공유 기능이 제한  
3 : CLI모드(다중 사용자 모드) → 제한이 없음, 마우스 지원X(ctrl+art시 복귀)  
4 : x  
5 : GUI모드(다중 사용자 모드) → 바탕화면(x-window)환경에서 사용하는 모드  
6 : 재부팅 모드
17. ls (목록 조회)  
특정 디렉터리 하위에 어떤 파일 목록이 있는지 출력하며 옵션 중복이 가능하다.  
ls  → 현재 디렉터리의 파일및 디렉터리 리스트 출력  
ls -l  → 파일의 자세한 정보를 리스트로 출력  
ls -a  →  숨김 파일까지 리스트로 출력  
ls -d  →  해당 디렉토리의 정보를 출력
18. pwd (현재 경로 확인)
19. cd (디렉토리 이동)  
cd /home  → home 디렉터리로 이동  
cd ..  →  현재 위치의 상위 디렉터리로 이동  
cd ~  →  계정의 홈 디렉터리로 이동
20. mkdir (디렉토리 생성)  
mkdir  →  빈 디렉토리를 생성  
mkdir -p test.txt/a  →  test.txt 폴더 안에 a 폴더 생성
21. rmdir (빈 디렉토리 삭제)  
rmdir -p a → a의 상위 디렉터리까지 삭제
22. touch (파일 생성)  
빈 파일 생성하는 명령어로 동일한 파일이 존재하지 않을 경우 빈 파일을 생성하며, 동일한 파일이 존재할 경우에는 지정한 파일의 시간 정보를 갱신한다.  
-> 단독으로 명령 불가능  
touch -t 희망하는 시간 text1 → 해당 파일을 원하는 시간 정보로 변경할 수 있다.
23. cat (파일 내용 출력)  
지정된 파일의 내용을 표준 출력  
-> 단독으로 명령 불가능  
-> 리다이렉션 사용으로 파일을 생성하여 내용을 입력할 수 있다.  
- 내용을 입력 후 ctrl d 로 내용 저장 후 종료
24. head (파일 내용 출력)  
파일의 처음부터 지정된 줄까지 출력  
default = 처음부터 10행  
#head option 파일이름  
25. tail (파일 내용 출력)  
파일의 마지막부터 지정된 줄까지 출력  
default = 마지막 부터 10행  
#tail option 파일이름  
-f : 실시간 추가되는 내용 출력
26. more (파일 내용 출력)  
많은 출력 결과를 한페이지씩 출력 -> 대용량 파일에서 원하는 데이터를 찾기 좋음  
| (파이프)를 이용해 다른 명령의 결과를 처리 할 수 있음 → 앞에 입력한 명령어 출력 결과에 추가적으로 명령어를 입력하는 방법  
#more 파일이름
27. less (파일 내용 출력)  
more와 유사 → less는 방향키로 페이지 이동 가능  
| 사용 가능  
28. grep (원하는 문자가 들어간 내용 출력)  
텍스트 파일이나 문자열이 많은 내용 중 원하는 패턴을 검색  
| 사용 가능  
#grep option *<원하는 패턴>* 파일이름  
<주요 옵션>  
 -i : 대소문자 무시  
 -v : 제외  
 -n : 줄 번호  
 -l : 파일명만  
 -r : 하위까지 검색  
 -w : 단어 일치  
 -c : 개수
29. cp (복사)  
원본 파일을 대상 파일로 복사  
windows는 복/붙 이 따로 작용하나, linux는 동시 작용  
cp -p = 소유주, 소속 그룹, 퍼미션, 시간 등을 동일하게 복사
30. mv (파이의 위치를 이동)  
파일의 위치를 이동하거나, 파일의 이름 변경  
중복이 없는 경우 파일 위치 변경과 동시에 이름을 다른 것으로 지정하면 변경이 가능  
#mv option 원본파일 대상파일
31. rm (삭제)  
파일 및 디렉터리 삭제  
rm option 파일 이름  
rm -r = 해당 디렉터리의 하위 디렉터리까지 모두 삭제  
rm -rf = 해당 디렉터리의 하위 디렉터리까지 경고 문구 없이 삭제  
⚠️ rm -rf는 매우 위험 (복구 불가)
32. file (파일의 종류 확인)  
파일의 종류를 자동으로 판단하여 출력 → 파일의 종류 정보를 표준 출력  
33. find (원하는 파일을 검색)  
지정된 디렉터리의 하위 파일 검색  
→ 조건(옵션)을 지정하고 해당 조건을 만족하는 형태의 파일들을 검색하고 그 결과로 해당 파일의 경로 정보를 표준출력  
#find *(디렉터리 경로)* option  
<주요 옵션>  
 find -empty : 비어있는 파일 찾기  
 find -name : 파일 이름으로 찾기  
  * → 모든 파일 / *의 위치에 따라 달라짐  
  ? → 모든 파일이라는 의미는 같으나 글자 수를 포함
  + not name  
 find -type (f/d/b…) : 지정된 파일 타입을 이용해 파일 찾기  
 find -mtime (+n/n/-n) : 파일 내용을 변경한 시간을 이용해 찾기(day 단위)  
 find -ctime (+n/n/-n) : 파일 정보를 변경한 시간을 이용해 찾기  
 finf -atime (+n/n/-n) : access(접속)시간을 이용해 찾기  
   (+n) : ~n일 동안 변경되지 않고, n일 전에 변경 된  
   (-n) : 최근 n일 동안 변경 된  
   ( n ) : 정확히 n일 전에 변경 된  
 find -exec <command> {} \; : find결과에 <command>적용  
 find -perm : 권한으로 검색  

>> 한 줄 정리 : 파일과 디렉토리를 관리하는 기본 명령어는 리눅스 사용의 핵심이다.
