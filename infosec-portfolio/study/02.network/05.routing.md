< 1️ > routing 
#  개요
라우팅은 네트워크 간 데이터를 목적지까지 전달하기 위해 최적의 경로를 선택하는 과정이다.

---

## 기본 개념
- 라우터는 목적지 IP를 확인하여 경로를 결정
- 라우팅 테이블을 참고하여 패킷 전달

---

## 라우팅 테이블

| 목적지 네트워크 | 게이트웨이 | 인터페이스 |

| 192.168.1.0 | 192.168.1.1 | eth0 |

---

## 라우팅 종류

### 정적 라우팅 (Static Routing)
- 관리자가 직접 경로 설정
- 단순하지만 유연성 낮음

---

### 동적 라우팅 (Dynamic Routing)
- 라우팅 프로토콜 사용
- 자동으로 경로 설정

예:
- RIP
- OSPF
- BGP

---

## 💡 특징
- 목적지까지 최적 경로 선택
- 네트워크 확장 시 필수

---

## 💭 느낀 점
라우팅은 단순 전달이 아니라, 효율적인 경로를 선택하는 중요한 과정이라는 것을 이해했다.

---

## 🧠 한 줄 정리
라우팅은 데이터를 목적지까지 전달하기 위한 경로 선택 과정이다.

< 2️ > nat
# 개요
NAT는 사설 IP 주소를 공인 IP 주소로 변환하여 외부 네트워크와 통신할 수 있도록 하는 기술이다.

---

## 기본 개념

- 내부 네트워크 → 사설 IP 사용
- 외부 인터넷 → 공인 IP 사용
- NAT 장비(라우터)가 주소 변환 수행

---

## NAT 종류

### Static NAT
- 1:1 주소 매핑
- 서버 운영에 사용

---

### Dynamic NAT
- 공인 IP 풀에서 할당

---

### PAT (NAT Overload)
- 하나의 공인 IP를 여러 사용자가 공유
- 포트 번호로 구분

---

## 💡 왜 필요한가?
- IPv4 주소 부족 해결
- 내부 네트워크 보호

---

예)
내부: 192.168.0.10 → 외부: 203.0.113.1:5000

---

## 💭 느낀 점
NAT를 통해 하나의 공인 IP로 여러 장치가 인터넷을 사용할 수 있다는 점이 인상적이었다.

---

## 🧠 한 줄 정리
NAT는 사설 IP를 공인 IP로 변환하여 인터넷 통신을 가능하게 한다.

< 3 > dhcp
# 개요
DHCP는 네트워크에 접속하는 장치에 IP 주소를 자동으로 할당해주는 프로토콜이다.

---

## 기본 개념

- IP 주소 자동 할당
- 네트워크 설정 자동 구성

---

## 동작 과정 (DORA)

1. Discover (요청)
2. Offer (제안)
3. Request (요청 수락)
4. Acknowledge (할당 완료)

---

## 💡 특징
- 수동 설정 필요 없음
- IP 충돌 방지
- 관리 효율성 증가

---

예)
PC → DHCP 서버 → IP 자동 할당

---

## 💭 느낀 점
DHCP를 통해 네트워크 설정이 자동으로 이루어지는 과정이 효율적이라고 느꼈다.

---

## 🧠 한 줄 정리
DHCP는 IP 주소를 자동으로 할당하는 프로토콜이다.
